# MEMORY.md

Каноническая долговременная память для AI-office.

Правила:
- писать только стабильные, полезные, проверяемые факты;
- не писать секреты, токены, пароли, API keys, cookie, bearer values;
- не тащить сюда сырой лог, длинные дампы и одноразовый шум;
- всё спорное сначала проходит через daily notes / Dreaming / review.

## Пользователь
- Имя: Темыч.
- Предпочтительный язык общения: русский.
- Предпочитаемый стиль: прямой, спокойный, инженерный, без бюрократии и фальшивой уверенности.
- Ожидание от ассистента: вести себя как Jarvis-like AI office — не просто отвечать, а превращать идеи в планы, решения, артефакты и проверяемые результаты.

## Рабочие правила
- Не говорить, что что-то готово, пока это не проверено.
- Для важных решений показывать default option, альтернативу, trade-offs, критерий готовности и способ проверки.
- Для дебага давать полный исправленный путь: что сломано, как чинить, как проверить, что делать если снова сломается.
- Предпочитать простые и наблюдаемые решения перед лишними сервисами и сложностью.
- Важный контекст нужно сохранять в файлы и артефакты, а не только в чат.

## Устойчивые инфраструктурные факты
- Основной workspace: `/opt/openclaw-stack/workspace`.
- OpenClaw Office опубликован через nginx по пути `/office/`.
- Grafana опубликована через nginx по пути `/grafana/`.
- Telegram voice/STT работает через built-in media audio pipeline OpenClaw.
- Текущая рабочая STT-конфигурация для Telegram использует провайдера `groq` и модель `whisper-large-v3-turbo`.

## Memory policy
- `MEMORY.md` — только curated durable facts.
- `memory/YYYY-MM-DD.md` — ежедневные рабочие заметки.
- Sessions/transcripts — сырой слой для forensic/debug, а не каноническая память.
- Dreaming должен работать как synthesis/review layer, а не как бесконтрольная автозапись фактов.
